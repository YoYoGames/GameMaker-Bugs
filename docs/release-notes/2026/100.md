## 2026.100.0 Release (IDE 1104 / GMS2 RT 1071) - [Changes](https://github.com/YoYoGames/GameMaker-Bugs/issues?q=is%3Aissue%20is%3Aclosed%20milestone%3A2026.1.0%20closed%3A2026-07-01..2026-07-22%20sort%3Aupdated-desc) (23/07/2026)

### Features Added Or Changed

- **GmlSpec: Rollback Functionality Removed [#15608](https://github.com/YoYoGames/GameMaker-Bugs/issues/15608)**
    - Be aware that this Betas-only functionality has now been removed entirely, now that there is no Monthly vs Betas distinction
    - If your game relies on rollback multiplayer, then please remain on 2026.0 until you have found an alternative system for your multiplayer

- **In-Game: Added Getter Functions For The Sequence Colour Setter Functions Added In 2026.0 [#15149](https://github.com/YoYoGames/GameMaker-Bugs/issues/15149)**
    - Added `layer_sequence_get_alpha()` and `layer_sequence_get_blend()` for getting a Sequence's current drawing colour values
    - A sample project and a little more info on how to use these two can be found on the issue

- **Package Manager: Validating And Repairing Packages [#14832](https://github.com/YoYoGames/GameMaker-Bugs/issues/14832)**
    - Added the ability for Package Manager to validate all your installed packages and repair/reinstall any ones it deems necessary, which should stop issues where a corrupted prefab can get "stuck"

- **Code Editor 2/Workspaces: Support For Splitting Windows Coming Soon**
    - Now, this feature is not enabled yet, as there are still some bugs to be resolved, but be aware that this version quietly has a number of the underlying changes to support a forthcoming ability to split workspace windows
    - This will soon give you the ability to do things like https://github.com/YoYoGames/GameMaker-Bugs/issues/13101
    - If you do happen to notice any new rendering issues, or problems working in full-workspace editors, like Sequences, Rooms, Scripts, etc., then please do report these asap!

## 2026.100.0 Release (IDE 1083 / GMS2 RT 1046) - [Changes](https://github.com/YoYoGames/GameMaker-Bugs/issues?q=is%3Aissue%20is%3Aclosed%20milestone%3A2026.1.0%20closed%3A<2026-07-02%20sort%3Aupdated-desc) (02/07/2026)

### Features Added Or Changed

- **Building Projects: Compile Errors Window Now Better Shows When An Error Came From A Prefab [#14835](https://github.com/YoYoGames/GameMaker-Bugs/issues/14835)**
    - Previously, prefabs and projects were treated the same way by the compile errors panel and so it was unclear whether a coding mistake came from the main project or a prefab you're using.
    - Now, you'll get a clear distinction and so a prefab error will look something like `Prefab dbg.test.error-1.0.0: Script: myScript1 at line 8 : Assignment operator expected` or `Prefab dbg.test.error-1.0.0: Object: myObject1 Collision Event With Object2 at line 1 : Assignment operator expected`

- **Prefab Library: Double-Clicking Any Asset Now "Previews" It [#14833](https://github.com/YoYoGames/GameMaker-Bugs/issues/14833)**
    - Double-clicking an asset now has the same result as right-clicking that asset and then selecting "Preview" from the content menu, and so double-clicking an asset will now open its viewer

- **Project Load: Show a choice dialog when a required prefab version can't be found, asking how to proceed so you can open the project [#15247](https://github.com/YoYoGames/GameMaker-Bugs/issues/15247)**
    - If your project uses a version of a prefab which is no longer available, then in previous IDEs you would need to manually raise the version of the prefab in the .yyp file outside GameMaker before you would be able to open that project
    - Now, this situation is detected and you are given the choice of removing the prefab link or to change the version that Package Manager will get for you to point at the newest version of that prefab
    - Once you have made your choice, GameMaker will then be able to open the project just fine again

- **UI/UX: Copy Call Stack Panel Contents To The OS Clipboard [#15244](https://github.com/YoYoGames/GameMaker-Bugs/issues/15244)**
    - There is now a new button which will copy the contents of the Call Stack panel to your OS clipboard as text that you can then paste somewhere outside GameMaker
